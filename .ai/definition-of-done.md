# Definition Of Done

## Default Bar

- No application source files were edited.
- If the task exposes a needed behavior change, the result hands off concrete implementation guidance to the lead developer instead of applying it.
- The narrowest relevant tests for the touched area have been run when validation is available.
- Docs or operator guidance are updated when behavior, commands, or expected workflow changed.
- Any skipped validation is stated plainly, with the remaining risk or follow-up gap.

## Not Assumed By Default

- Do not claim lint passed; no lint script is configured.
- Do not claim formatting ran as a standard repo command; no format script is configured.
- Do not require full-suite tests, compile, or build unless the specific task justifies the wider check.
